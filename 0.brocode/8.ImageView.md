# 📘 JavaFX Lesson: Displaying an Image with ImageView

## 🎯 Goal of This Lesson

Learn how to:

- Load an image from your resources
    
- Display it inside an `ImageView`
    
- Trigger the action using a button
    

This is the foundation for building photo viewers, galleries, games, and UI graphics.

---

# 🧩 1. Controller.java

```java
package application;

import javafx.fxml.FXML;
import javafx.scene.control.Button;
import javafx.scene.image.Image;
import javafx.scene.image.ImageView;

public class Controller {

    // ImageView = a UI element that displays images
    // Image     = the actual picture file

    @FXML
    ImageView myImageView;
    @FXML
    Button myButton;

    // Load image from project resources
    Image myImage = new Image(getClass().getResourceAsStream("shrek2.jpg"));

    public void displayImage() {
        myImageView.setImage(myImage);
    }
}
```

---

# 🧠 2. Key Concepts Explained

## 🔹 2.1 Image and ImageView

### **Image**

Represents the _actual picture file_.  
You load it from your project using:

```java
new Image(getClass().getResourceAsStream("shrek2.jpg"));
```

This loads the picture from the **same folder/package** as your Controller.

### **ImageView**

A UI Node that **shows the Image** on the screen.

Think of it like:

- **Image** = the photo
    
- **ImageView** = the photo frame
    

---

## 🔹 2.2 @FXML Bindings

You declare:

```java
@FXML ImageView myImageView;
@FXML Button myButton;
```

These connect to your FXML file, like:

```xml
<ImageView fx:id="myImageView" />
<Button fx:id="myButton" onAction="#displayImage" />
```

---

# 🎮 3. How the Button Works

When the user clicks the button, JavaFX calls:

```java
public void displayImage()
```

This method updates the UI:

```java
myImageView.setImage(myImage);
```

Which instantly shows your picture on screen.

---

# 📂 4. Folder Structure (Very Important)

Your files MUST be in the same package:

```
src/
 └── application/
        ├── Main.java
        ├── Controller.java
        ├── Main.fxml
        ├── application.css
        └── shrek2.jpg
```

If the image is **not** in the same folder → JavaFX won’t load it.

---

# 🧪 5. Common Errors & Fixes

### ❌ NullPointerException for ImageView

Means the fx:id doesn’t match your FXML.

### ❌ File not loading

Check:

```java
getClass().getResourceAsStream("shrek2.jpg")
```

→ The file must be inside the same folder as Controller.java.

### ❌ “Input stream must not be null!”

This means the image path is wrong.
