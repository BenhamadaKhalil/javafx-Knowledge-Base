# 🎓 **JavaFX Lesson: Communication Between Scenes (Passing Data Between Controllers)**

In JavaFX, it’s very common to have multiple screens (scenes).  
For example:

- Scene 1: Login screen
    
- Scene 2: Dashboard
    
- Scene 3: Profile page
    

Very often, you need to **send data from one scene to another** — like sending the username from the login screen to the home screen.

This lesson explains exactly **how to pass data between controllers** using your code.

---

# 🧩 **1. The Concept: Why we need communication**

Consider this sequence:

1. User types their name in Scene1 (login screen)
    
2. User clicks the login button
    
3. Scene should change to Scene2
    
4. Scene2 must display the user’s name
    

This means Scene1 must send the username to Scene2 **before Scene2 appears**.

---

# 🧠 **2. Why FXMLLoader is the key**

Normally to load a scene, many beginners write:

```java
Parent root = FXMLLoader.load(getClass().getResource("Scene2.fxml"));
```

❌ But this line **does not** let you access the controller.  
You cannot call methods on the controller.

To communicate between scenes, you must use:

```java
FXMLLoader loader = new FXMLLoader(getClass().getResource("Scene2.fxml"));
Parent root = loader.load();
Scene2Controller controller = loader.getController();
```

This gives you access to Scene2’s controller.

---

# 🛠️ **3. Scene1Controller: Sending Data to Scene2**

```java
public void login(ActionEvent event) throws IOException {

    String username = nameTextField.getText();

    FXMLLoader loader = new FXMLLoader(getClass().getResource("Scene2.fxml"));
    root = loader.load();

    Scene2Controller scene2Controller = loader.getController();
    scene2Controller.displayName(username);

    stage = (Stage)((Node)event.getSource()).getScene().getWindow();
    scene = new Scene(root);
    stage.setScene(scene);
    stage.show();
}
```

### Let’s break this down:

---

## ✔ Step 1: Get the user input

```java
String username = nameTextField.getText();
```

We read the text the user typed.

---

## ✔ Step 2: Load Scene2 manually using FXMLLoader

```java
FXMLLoader loader = new FXMLLoader(getClass().getResource("Scene2.fxml"));
root = loader.load();
```

This is the **correct way** to load an FXML file when you want to communicate with its controller.

---

## ✔ Step 3: Access Scene2’s controller

```java
Scene2Controller scene2Controller = loader.getController();
```

This gives us an object representing Scene2’s controller.

---

## ✔ Step 4: Pass the data to Scene2

```java
scene2Controller.displayName(username);
```

Now Scene2 receives the username.

---

## ✔ Step 5: Switch to Scene2 normally

```java
stage = (Stage)((Node)event.getSource()).getScene().getWindow();
scene = new Scene(root);
stage.setScene(scene);
stage.show();
```

---

# 🧩 **4. Scene2Controller: Receiving the Data**

Scene2Controller:

```java
public class Scene2Controller {

    @FXML
    Label nameLabel;

    public void displayName(String username) {
        nameLabel.setText("Hello: " + username);
    }
}
```

### What happens here?

✔ `displayName()` receives the username  
✔ It updates the UI label  
✔ JavaFX automatically updates the screen when the scene loads

---

# 📌 Important: `@FXML` must be used

```java
@FXML
Label nameLabel;
```

This connects the Java variable to the Label in Scene2.fxml.

Without `@FXML`, the label will be `null`.

---

# 🎨 **5. How Scene2.fxml works with the controller**

Scene2.fxml includes:

```xml
fx:controller="application.Scene2Controller"
```

Meaning this FXML works with Scene2Controller.java.  
So when JavaFX loads Scene2, it creates the controller and connects the UI elements.

---

# 🧱 **6. Summary of Communication Steps**

Here’s the whole idea in the simplest form:

---

## 🔹 Scene1Controller (Sender)

1. Load Scene2 with `FXMLLoader`
    
2. Get controller using `loader.getController()`
    
3. Call a method on it and pass data
    
4. Switch scenes
    

---

## 🔹 Scene2Controller (Receiver)

1. Create a method like `displayName()`
    
2. Accept the sent data
    
3. Update UI elements
    

---
