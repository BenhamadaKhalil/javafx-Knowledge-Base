## 🎓 Lesson 2 – Setting Up IntelliJ IDEA and Building Your First JavaFX App

### 🧩 Overview

In this lesson, we’ll:

- Install and configure **IntelliJ IDEA** for JavaFX development.
    
- Create a **new JavaFX project**.
    
- Understand key files (`Main.java`, `HelloApplication.java`, `hello-view.fxml`, etc.).
    
- Introduce **FXML**, **Scene Builder**, and **CSS styling**.
    

---

### 🧠 1. Installing IntelliJ IDEA

1. Visit [jetbrains.com/idea](https://www.jetbrains.com/idea/).
    
2. You’ll find **two editions**:
    
    - **Community Edition** (Free)
        
    - **Ultimate Edition** (Paid)
        

> 💡 The **Community Edition** is enough for JavaFX development — all core features are available.

3. Download and install IntelliJ.  
    It’s around **600 MB** and installs like any regular app.
    

---

### ⚙️ 2. Creating a New JavaFX Project

Once IntelliJ is ready:

1. Click **New Project** → select **JavaFX** from the list.
    
2. Name your project, for example:
    
    ```
    FirstJavaFXApp
    ```
    
3. (Optional) Customize the **Group ID** and **Artifact ID** — these are identifiers used by Maven.
    
    - **Group ID:** usually your domain name in reverse.  
        Example: `com.example`
        
    - **Artifact ID:** your project name, e.g. `javafx-demo`.
        

> 🧩 Example  
> If your domain is `example.com`, your Group ID becomes `com.example`.

4. Click **Next**, then **Finish**. IntelliJ will set up a ready-to-run JavaFX project.
    

---

### 📦 3. Understanding the Project Structure

When the project is created, IntelliJ will generate:

```
src/
 └── main/
     ├── java/
     │   └── com/example/javafxdemo/
     │        ├── HelloApplication.java
     │        └── HelloController.java
     └── resources/
          └── com/example/javafxdemo/
               └── hello-view.fxml
pom.xml
```

#### 🧩 Key Files:

|File|Description|
|---|---|
|**HelloApplication.java**|Contains the `start()` method — the entry point of your JavaFX app.|
|**HelloController.java**|Handles logic and user interaction (event handling).|
|**hello-view.fxml**|Defines the **UI layout** in XML-like syntax.|
|**pom.xml**|Maven configuration file that manages dependencies and libraries.|

---

### 🪄 4. Running the App

- Wait while IntelliJ **indexes and downloads** dependencies (can take up to a minute).
    
- Once complete, click **Run ▶️**.
    
- You’ll see a simple **JavaFX window** with a **“Hello!”** button.
    

---

### 🧰 5. Working with FXML

JavaFX provides **two main ways** to create interfaces:

1. **Pure Java Code:**  
    Write everything manually using Java objects (like Swing).
    
2. **FXML (recommended):**  
    XML-like files that describe the interface structure.  
    Example:
    
    ```xml
    <Button text="Hello!" />
    ```
    
    This separates **design** from **logic**, similar to HTML + JS in web apps.
    

> 💡 If you’ve used HTML before, FXML will feel very familiar.

---

### 🧱 6. Using Scene Builder (GUI Design Tool)

**Scene Builder** lets you **drag and drop** UI elements (buttons, labels, fields) to design your layout visually.

#### 🔧 Steps:

1. Download **Scene Builder** from [gluonhq.com/products/scene-builder](https://gluonhq.com/products/scene-builder/).
    
2. Open your `.fxml` file with Scene Builder.
    
3. Add controls (buttons, labels, text fields, etc.) from the **Components** panel.
    
4. Save — your FXML updates automatically.
    

> 🖱️ “What You See Is What You Get” — the final UI looks just like what you design.

---

### 🎨 7. Styling with CSS

Just like web development, JavaFX supports **CSS** for styling.

You can:

- Change colors, fonts, borders, and more.
    
- Create reusable style files (e.g., `style.css`).
    

Example:

```css
.button {
  -fx-background-color: #3498db;
  -fx-text-fill: white;
  -fx-font-size: 14px;
}
```

Attach your stylesheet in Java or FXML:

```java
scene.getStylesheets().add("style.css");
```
