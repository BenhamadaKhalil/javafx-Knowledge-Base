## 🎓 Lesson 2 – Setting Up the JavaFX Development Environment

### 🧩 Goal of This Lesson

In this lesson, we’ll:

- Set up the **JavaFX development environment**.
    
- Use **IntelliJ IDEA** to create and run our first JavaFX project.
    
- Understand **FXML**, **Scene Builder**, and **CSS** integration in JavaFX.
    

---

### ⚙️ 1. Installing IntelliJ IDEA

We’ll be using **IntelliJ IDEA** as our main IDE (Integrated Development Environment) throughout this course.

#### 🔹 Step 1 – Download IntelliJ

- Go to **[jetbrains.com/idea](https://www.jetbrains.com/idea/)**.
    
- Download **IntelliJ IDEA Community Edition** (the free version).
    

> 💡 The _Ultimate Edition_ offers more features, but for JavaFX, the _Community Edition_ is more than enough.

#### 🔹 Step 2 – Install

- The installer is around **600 MB**.
    
- The installation process is straightforward — follow the prompts and open IntelliJ once done.
    

---

### 🧱 2. Creating Your First JavaFX Project

#### 🪄 Step 1 – Start a New Project

- Open IntelliJ IDEA → **New Project**
    
- From the project types, select **JavaFX**.
    

#### 🪄 Step 2 – Configure Project Info

You’ll see:

- **Group ID** → a reverse domain identifier (e.g., `com.example`)
    
- **Artifact ID** → your project name
    

Example:

```
Group ID: com.javamasters
Artifact ID: javafx-firstapp
```

> 💬 You can keep the default values or modify them — they mainly help in identifying your project uniquely.

#### 🪄 Step 3 – Choose Dependencies

You may see options like:

- **BootstrapFX**, **ControlsFX**, **FormsFX**, **TilesFX**, etc.  
    These are _optional libraries_ that add extra components.  
    👉 For now, **leave them unchecked** — we’ll learn **pure JavaFX** first.
    

#### 🪄 Step 4 – Finish Setup

Click **Create**. IntelliJ will:

- Download dependencies
    
- Index the JDK
    
- Set up your JavaFX template
    

This might take **a few seconds to a minute**.

---

### 💻 3. Exploring the JavaFX Project Structure

When the project opens, you’ll see a few key files:

|File|Description|
|---|---|
|`Main.java`|The main class – contains `main()` and `start()` methods.|
|`module-info.java`|Manages your JavaFX modules and dependencies.|
|`.fxml` file|Defines your UI layout using FXML markup.|

You can customize IntelliJ’s theme (light or dark) — choose whichever helps you focus better.

---

### 🧠 4. Two Ways to Build a JavaFX UI

JavaFX offers **two approaches** to create your user interface:

1. **Pure Java Code** – build the UI programmatically.
    
    ```java
    Button btn = new Button("Click Me");
    root.getChildren().add(btn);
    ```
    
2. **FXML (JavaFX XML)** – build the UI with markup similar to HTML.
    
    ```xml
    <Button text="Click Me" />
    ```
    

> 💬 Most developers prefer **FXML**, as it separates design from logic and is easier to visualize.

---

### 🎨 5. Introducing Scene Builder

To make designing FXML easier, use **Scene Builder**.

#### 🔹 What Is Scene Builder?

Scene Builder is a drag-and-drop UI tool for JavaFX.

- Download it from: [https://gluonhq.com/products/scene-builder/](https://gluonhq.com/products/scene-builder/)
    
- It allows you to design your layout **visually** (no coding required).
    

#### 🔹 How It Works

- Open your `.fxml` file with Scene Builder.
    
- Drag components (buttons, text fields, combo boxes, etc.) onto the layout.
    
- Save the file — it automatically updates the FXML code.
    

This is a **WYSIWYG (What You See Is What You Get)** design tool — the interface you design is exactly what users will see.

### 🧵 6. Styling with CSS

JavaFX supports **CSS styling** just like web development.

You can:

- Change **colors**, **fonts**, **sizes**, and **backgrounds**.
    
- Create and link `.css` files to your JavaFX project.
    
- Use standard CSS syntax.
    

Example:

```css
.button {
    -fx-background-color: #3498db;
    -fx-text-fill: white;
    -fx-font-size: 16px;
}
```

> 💡 If you’ve done web development before, CSS in JavaFX will feel familiar.
