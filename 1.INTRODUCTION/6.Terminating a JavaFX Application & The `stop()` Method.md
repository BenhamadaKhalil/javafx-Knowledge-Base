# 🎬 Lesson 4 – Terminating a JavaFX Application & The `stop()` Method

## 🧠 Overview

In this lesson, we’ll learn **how to properly close (terminate)** a JavaFX application — both **explicitly** and **implicitly**.  
We’ll also explore the **`stop()`** method in detail, which is automatically called whenever your application is closed.

---

## ⚙️ Two Ways to Exit a JavaFX Application

JavaFX provides **two ways** to terminate your app:

|Type|Description|
|---|---|
|🔹 **Explicit Termination**|You, the developer, manually tell JavaFX to exit.|
|🔸 **Implicit Termination**|The app exits automatically when the user closes the last window.|

---

## 🧩 1️⃣ Explicit Exit – Using `Platform.exit()`

The **explicit** way to close a JavaFX app is by calling the static method:

```java
Platform.exit();
```

This method belongs to the **`javafx.application.Platform`** class.

### 🧪 Example

```java
Button exitButton = new Button("Exit Explicitly");
exitButton.setOnAction(e -> Platform.exit());
```

✅ When you click this button:

1. The application begins closing.
    
2. The **`stop()`** method (if defined) is called.
    
3. The application terminates completely.
    

---

## 🧩 2️⃣ Implicit Exit – Closing the Window

By default, when you **click the close (X)** button on your app window:

- JavaFX automatically stops the application.
    
- It **calls the `stop()` method** before exiting.
    

You don’t need to write any special code for this — it happens **implicitly**.

---

## 🧠 The `stop()` Method Explained

When a JavaFX app shuts down (explicitly or implicitly), the **`stop()`** method is automatically executed.

### 🧱 Definition

```java
@Override
public void stop() {
    System.out.println("Closing the application...");
}
```

- It’s called **before** the application fully exits.
    
- It’s where you put **cleanup logic**, such as:
    
    - Saving user data
        
    - Closing database connections
        
    - Releasing resources
        

---

## 🧵 Which Thread Calls `stop()`?

Let’s check which thread runs this method:

```java
@Override
public void stop() {
    System.out.println("stop() → " + Thread.currentThread().getName());
}
```

🧩 Output:

```
stop() → JavaFX Application Thread
```

✅ This means:

- The `stop()` method runs on the **JavaFX Application Thread**,  
    not the Launcher Thread.
    

Among the lifecycle methods:

|Method|Thread|
|---|---|
|Constructor|Application Thread|
|`init()`|Launcher Thread|
|`start()`|Application Thread|
|`stop()`|Application Thread|

---

## 🧪 Example: Explicit Exit with `stop()`

```java
@Override
public void start(Stage stage) {
    Button exitButton = new Button("Exit Explicitly");
    exitButton.setOnAction(e -> Platform.exit());

    VBox root = new VBox(exitButton);
    Scene scene = new Scene(root, 300, 200);
    stage.setScene(scene);
    stage.show();
}

@Override
public void stop() {
    System.out.println("Closing the application...");
    System.out.println("Thread: " + Thread.currentThread().getName());
}
```

🧩 Output when clicking the button:

```
Closing the application...
Thread: JavaFX Application Thread
```

## ⚙️ The `Platform.setImplicitExit(boolean)` Method

JavaFX automatically exits when **all windows are closed** — this is called **implicit exit**.  
But you can change that behavior using:

```java
Platform.setImplicitExit(false);
```

### 💡 What Happens?

|Setting|Behavior|
|---|---|
|`true` (default)|When you close all windows, the app **exits**.|
|`false`|When you close all windows, the **app keeps running** in the background.|

This is useful when:

- Your app has **multiple windows**.
    
- You want to **hide** a window temporarily (e.g., system tray apps).
    

---

### 🧪 Example

```java
@Override
public void start(Stage stage) {
    Platform.setImplicitExit(false); // Prevent automatic exit

    Button closeWindow = new Button("Close Window");
    closeWindow.setOnAction(e -> stage.close());

    Scene scene = new Scene(new VBox(closeWindow), 300, 200);
    stage.setScene(scene);
    stage.show();
}
```

🧩 Behavior:

- The window closes, but the **application keeps running**.
    
- The process stays active until you call `Platform.exit()` manually.
    

---

## 🧠 Summary

|Concept|Description|
|---|---|
|`Platform.exit()`|Explicitly closes the app and triggers `stop()`.|
|`stop()`|Lifecycle method called before the app terminates.|
|Implicit Exit|App exits automatically when all windows are closed (default behavior).|
|`Platform.setImplicitExit(false)`|Prevents the app from closing when windows are closed.|

---

## 💬 Key Takeaways

- Always use **`stop()`** for cleanup and shutdown logic.
    
- Both **explicit** and **implicit** exits trigger the **`stop()`** method.
    
- `stop()` runs on the **JavaFX Application Thread**.
    
- `Platform.setImplicitExit(false)` keeps the app running even after all windows are closed.
    
- Most apps use the **implicit exit** (default) behavior.
