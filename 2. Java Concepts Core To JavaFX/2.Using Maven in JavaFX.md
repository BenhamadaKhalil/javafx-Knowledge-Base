# 🎬 Lesson – Using Maven in JavaFX

## 🧠 Overview

This section introduces **Maven**, a **project management and build automation tool**, and demonstrates how to integrate it with a JavaFX project.

---

## 🔑 What is Maven?

- Maven manages a project’s **entire lifecycle**, including:
    
    - **Building** the project
        
    - **Testing**
        
    - **Deploying**
        
    - **Documentation and reporting**
        
- Maven simplifies **project builds** and manages **different versions** in a **standardized way**.
    
- It's particularly useful when a project **grows in complexity** or involves **multiple developers/teams**.
    

---

## ⚡ Key Concepts

### 1. Convention over Configuration

- Maven favors **convention** rather than requiring heavy manual configuration.
    
- Compared to Gradle:
    
    - **Gradle:** highly flexible, customizable
        
    - **Maven:** opinionated, standardized, easier to learn for beginners
        

### 2. Project Structure

Maven generates a **default folder structure**:

```
project-root/
│
├─ src/main/java      # Java source files
├─ src/main/resources # Resources (images, XML files, etc.)
├─ src/test/java      # Unit tests
└─ pom.xml            # Project Object Model (configuration file)
```

### 3. `pom.xml` (Project Object Model)

- XML file describing project **configuration, dependencies, and plugins**.
    
- Key sections:
    
    - **Properties:** define project variables (e.g., Java version)
        
    - **Dependencies:** external libraries required by the project
        
    - **Plugins:** extend Maven functionality for building, running, or packaging
        

---

## 🛠 Setting Up a JavaFX Project with Maven

### 1. Create a Maven Project

- Choose **Maven** → **Next**
    
- Set project name and **artifact coordinates**
    
    - Example: `com.javafx.masterclass:mavenfx`
        

### 2. Add Dependencies

- Example dependencies for JavaFX:
    
    - `javafx-controls`
        
    - `javafx-fxml`
        

```xml
<dependencies>
    <dependency>
        <groupId>org.openjfx</groupId>
        <artifactId>javafx-controls</artifactId>
        <version>15.0.1</version>
    </dependency>
    <dependency>
        <groupId>org.openjfx</groupId>
        <artifactId>javafx-fxml</artifactId>
        <version>15.0.1</version>
    </dependency>
</dependencies>
```

- **Tip:** Only include dependencies you **actually use**.
    

### 3. Add Plugins

- Common Maven plugins:
    
    - **Compiler plugin** – sets Java version
        
    - **JavaFX Maven plugin** – runs JavaFX apps
        

```xml
<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-compiler-plugin</artifactId>
      <version>3.10.1</version>
      <configuration>
        <source>17</source>
        <target>17</target>
      </configuration>
    </plugin>

    <plugin>
      <groupId>org.openjfx</groupId>
      <artifactId>javafx-maven-plugin</artifactId>
      <version>0.0.8</version>
      <configuration>
        <mainClass>com.c.mavenfx.app.App</mainClass>
      </configuration>
    </plugin>
  </plugins>
</build>
```

---

## 📦 Java Module System (Brief Overview)

- Organizes code into **modules** to encapsulate packages.
    
- Example: `module-info.java`
    

```java
module com.c.mavenfx.app {
    requires javafx.controls;
    requires javafx.fxml;
    opens com.c.mavenfx.app to javafx.fxml;
}
```

---

## 🏃 Running the Project

1. Ensure Maven changes are **loaded** in IDE.
    
2. Create a basic JavaFX **application class**:
    

```java
package com.c.mavenfx.app;

import javafx.application.Application;
import javafx.scene.Scene;
import javafx.scene.control.Label;
import javafx.scene.layout.VBox;
import javafx.stage.Stage;

public class App extends Application {
    @Override
    public void start(Stage stage) {
        VBox root = new VBox();
        root.setAlignment(javafx.geometry.Pos.CENTER);
        root.getChildren().add(new Label("Hello, World!"));

        Scene scene = new Scene(root, 400, 300);
        stage.setTitle("MavenFX");
        stage.setScene(scene);
        stage.show();
    }

    public static void main(String[] args) {
        launch();
    }
}
```

3. Run the application via **Maven plugin** or IDE run configuration.
