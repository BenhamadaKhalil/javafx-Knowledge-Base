## **Factory Pattern in Java**

### **1. Purpose**

The **factory pattern** allows you to create objects **without exposing the creation logic** to the user.

- Useful when an object can be created in multiple ways.
    
- Helps **simplify object creation** and hides complexity.
    

---

### **2. Example Setup**

**Interface**: `Shape`

```java
public interface Shape {
    void draw();
}
```

**Concrete implementations**:

```java
public class Rectangle implements Shape {
    public void draw() {
        System.out.println("This is a rectangle.");
    }
}

public class Circle implements Shape {
    public void draw() {
        System.out.println("This is a circle.");
    }
}

public class Square implements Shape {
    public void draw() {
        System.out.println("This is a square.");
    }
}
```

---

### **3. Factory Class**

```java
public class ShapeFactory {
    
    public static Shape getRectangle() {
        return new Rectangle();
    }

    public static Shape getCircle() {
        return new Circle();
    }

    public static Shape getSquare() {
        return new Square();
    }
}
```

**Usage**:

```java
Shape rect = ShapeFactory.getRectangle();
rect.draw();  // Output: This is a rectangle.

Shape circle = ShapeFactory.getCircle();
circle.draw();  // Output: This is a circle.
```

---

### **4. Why Use an Interface**

- Implements the **Open/Closed Principle**:
    
    - **Open for extension**: You can add new shapes (e.g., `Triangle`) without modifying existing code.
        
    - **Closed for modification**: Existing shapes (`Rectangle`, `Circle`) do not need to be changed.
        
- Avoids **breaking the codebase** when new shapes are added.
    

**Example**: Adding a new `Triangle`:

```java
public class Triangle implements Shape {
    public void draw() {
        System.out.println("This is a triangle.");
    }
}
```

- No need to modify `Rectangle`, `Circle`, or `Square`.
    

---

### **5. Benefits**

1. Simplifies object creation.
    
2. Hides complex creation logic.
    
3. Makes the code **extensible** without modifying existing classes.
    
4. Compatible with **JavaFX** patterns, e.g., for creating observable collections.
