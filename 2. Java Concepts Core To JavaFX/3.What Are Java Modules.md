## **1. What Are Java Modules?**

- Modules are **groups of related packages**, forming a new **packaging system in Java**.
    
- Introduced in **Java 9**.
    
- Simplifies:
    
    - Distribution of applications
        
    - Encapsulation of packages
        
    - Dependency management
        

---

## **2. Benefits of Using Modules**

1. **Better Distribution**
    
    - Packages needed for the application are grouped into modules.
        
    - APIs can be deployed in separate modules.
        
    - Only necessary parts are included for other modules.
        
2. **Better Encapsulation**
    
    - The `module-info.java` file allows specifying:
        
        - **Visible packages** (`exports`)
            
        - **Private packages**
            
    - Similar to `public` and `private` fields in a class.
        
    - Helps **detect missing classes/packages** early.
        
3. **Robust Packaging**
    
    - By default, packages are private.
        
    - `exports` and `opens` keywords control visibility:
        
        - `exports` → make packages accessible to other modules.
            
        - `opens` → allow reflective access (e.g., FXML in JavaFX).

## **3. JavaFX Project Without Maven**

### **Steps:**

1. Download **JavaFX SDK** from the official website.
    
2. Create a **new Java project** in the IDE.
    
3. Add the downloaded SDK as a **library**.
    
4. Create a module descriptor `module-info.java`:
    
    ```java
    module com.example.moduleseffects {
        requires javafx.controls;
        requires javafx.fxml;
        exports com.example.moduleseffects;
        opens com.example.moduleseffects to javafx.fxml;
    }
    ```
    
5. Create main JavaFX class:
    
    ```java
    public class MainApp extends Application {
        @Override
        public void start(Stage stage) {
            Label label = new Label("Hello World");
            StackPane root = new StackPane(label);
            Scene scene = new Scene(root, 400, 300);
            stage.setTitle("Modules Effects");
            stage.setScene(scene);
            stage.show();
        }
    
        public static void main(String[] args) {
            launch(args);
        }
    }
    ```
    

---

## **4. Important Notes**

- **Requires** → declares modules needed.
    
- **Exports** → makes a package available to other modules.
    
- **Opens** → makes a package accessible for reflection (FXML linkage).
    
- Manual dependency management can become **error-prone**.
    
- **Maven/Gradle** recommended:
    
    - Automatically handles dependencies and transitive dependencies.
        
    - Essential for large projects.
