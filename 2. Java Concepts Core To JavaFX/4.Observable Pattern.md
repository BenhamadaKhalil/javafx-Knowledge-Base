## **1. What is a Design Pattern?**

- A **design pattern** is a reusable solution to a common problem in software design.
    
- The **Observable Pattern** is one such design pattern that focuses on communication between objects.
    

---

## **2. Problem Observable Pattern Solves**

- Imagine you have an object or group of objects (observers) interested in the **state of another object** (observable).
    
- Whenever the observable’s state changes, the observers need to be notified automatically.
    

**Example in JavaFX:**

- A list of items in memory (e.g., `ArrayList`) is displayed in the UI.
    
- If the list changes (items added/removed), the UI must update automatically.
    

---

## **3. Observable Pattern Concept**

- The **observable object** keeps track of a list of subscribers (observers).
    
- When the state changes, the observable **notifies all subscribers** by calling a method like `update()`.
    

---

## **4. Implementation Steps in Java**

### **Step 1: Create the Observable Class**

```java
public class NewsAgency {
    private String news; // state to be observed
    private final List<Channel> channels = new ArrayList<>(); // observers

    public NewsAgency(String news) {
        this.news = news;
    }

    // Add a subscriber
    public void addObserver(Channel channel) {
        channels.add(channel);
    }

    // Remove a subscriber
    public void removeObserver(Channel channel) {
        channels.remove(channel);
    }

    // Update state and notify observers
    public void setNews(String news) {
        this.news = news;
        for (Channel channel : channels) {
            channel.update(this.news);
        }
    }
}
```

---

### **Step 2: Create the Observer Interface**

```java
public interface Channel {
    void update(String news); // method called when observable state changes
}
```

---

### **Step 3: Create Concrete Observer**

```java
public class NewsChannel implements Channel {
    private String news;

    @Override
    public void update(String news) {
        this.news = news;
        System.out.println("Breaking News: " + news);
    }
}
```

---

### **Step 4: Use in Main Class**

```java
public class Main {
    public static void main(String[] args) {
        NewsAgency agency = new NewsAgency("No news");
        NewsChannel channel1 = new NewsChannel();
        NewsChannel channel2 = new NewsChannel();

        agency.addObserver(channel1);
        agency.addObserver(channel2);

        agency.setNews("Elections have started!");
    }
}
```

**Output:**

```
Breaking News: Elections have started!
Breaking News: Elections have started!
```

---

## **5. How It Relates to JavaFX**

- Instead of printing, the observers could be UI components (labels, lists, buttons).
    
- When the observable state changes, the UI automatically updates.
    
- Helps separate **business logic** from **UI logic** efficiently.
    

---

✅ **Key Points:**

1. Observable pattern **decouples objects**: The observable doesn't need to know about the implementation of observers.
    
2. Observers **subscribe** to an observable and get **notified automatically** when state changes.
    
3. Great for **dynamic UIs**, reactive programming, and **real-time updates**.
